# Git Repo Reference RU

Этот файл фиксирует опорный GitHub-репозиторий проекта и правило работы с удалёнными путями в документации.

## Репозиторий проекта

- Git repository URL:
  - `https://github.com/cncmolot-wq/MatrixClockS3.git`
- Web repository root:
  - `https://github.com/cncmolot-wq/MatrixClockS3`

## Назначение

- Этот файл нужен как опорная ссылка для сверки путей в локальной документации.
- Он не предназначен для автоматического "перепридумывания" структуры GitHub-репозитория.
- Он нужен, чтобы локальные документы сверялись с реальным опубликованным репозиторием, а не с предположениями.

## Правило

- GitHub-репозиторий и его удалённая структура не редактируются по предположению.
- Допустимо только:
  - сверять локальные документные пути с реальным репозиторием;
  - исправлять локальные документы, если они расходятся с фактической опубликованной структурой.
- Недопустимо:
  - придумывать новую удалённую структуру без подтверждения;
  - молча менять remote URL в локальных документах без сверки;
  - считать локальную структуру `docs/...` автоматически равной удалённой GitHub-структуре.

## Разделение путей

- Локально проект живёт через:
  - `docs/project/...`
  - `docs/chatgpt_git/...`
- Удалённо GitHub может публиковать:
  - `project/...`
  - `chatgpt_git/...`

Это не ошибка само по себе. Ошибка возникает тогда, когда локальные и удалённые пути смешиваются в одном ссылочном слое без явного правила.

## Практика

- Перед правкой GitHub-ссылок в локальной документации сначала сверяется фактический репозиторий:
  - структура папок;
  - реальные web paths;
  - имена файлов.
- Только после этого исправляются локальные документы.

## Публикация

- Локальные документы правятся внутри проекта.
- Обновление GitHub-репозитория выполняет только пользователь.
- После локальных правок пользователь сам перезаливает опубликованные папки:
  - `chatgpt_git`
  - `project`
- Локальная правка не считается автоматически опубликованной.
- Внешний ChatGPT должен ориентироваться только на ту версию документов, которую пользователь реально выложил в GitHub.

- Пользователь добавил скриншот GitHub-репозитория как наглядный пример текущей опубликованной структуры.
- Локальная копия этого примера хранится здесь:
  - `docs/project/GIT_REPO_STRUCTURE_EXAMPLE.png`
- Если локальные документы расходятся с этой подтверждённой структурой, корректируются локальные документы, а не "воображаемая" структура репозитория.
