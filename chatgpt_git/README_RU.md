# ChatGPT + GitHub Package

Этот каталог — внешний publish-ready пакет для обычного ChatGPT.

Его задача:
- дать внешнему ChatGPT короткий управляемый набор правил;
- дать стабильные удалённые ссылки на русскую документацию проекта;
- не смешивать внешний пакет с живой внутренней документацией проекта.

## Что лежит в папке

- `mc_core.v4.ini`
  Внешний машинно-ориентированный профиль для draft-режима.
- `CHATGPT_PROJECT_SETTINGS_RU.md`
  Готовый текст для вставки в `Project Instructions` обычного ChatGPT.
- `CHATGPT_TASK_CAPSULE_RU.md`
  Шаблон узкой задачи для внешнего ChatGPT.
- `GITHUB_PUBLISH_STEPS_RU.md`
  Пошаговая инструкция публикации и проверки ссылок.

## Важное разделение

Локальные исходники проекта живут здесь:
- `docs/project/...`
- `docs/chatgpt_git/...`

Но удалённые ссылки для внешнего ChatGPT в этом пакете указывают на опубликованные GitHub-пути:
- `project/...`
- `chatgpt_git/...`

То есть:
- локальная структура нужна для работы внутри проекта;
- удалённая структура нужна для внешнего ChatGPT после публикации в GitHub.

## Что этот пакет не заменяет

Этот пакет не заменяет:
- `docs/project/README_RU.md`
- `docs/project/REFACTOR_PLAN_RU.md`
- `docs/project/PROJECT_EDIT_RULES.md`

Они остаются живым внутренним источником истины проекта.

## Рабочая схема

1. Ты публикуешь содержимое этого внешнего пакета в GitHub.
2. Ты публикуешь русские проектные документы в удалённый набор `project/...`.
3. Внешний ChatGPT получает:
   - project settings
   - task capsule
   - remote docs links
4. Если внешний ChatGPT не может открыть ссылки, он обязан работать только по встроенному `Docs summary`.
5. Финальная интеграция, проверка, сборка и синхронизация документации всё равно остаются за Codex.

## Принцип

Внешний ChatGPT:
- не архитектор;
- не финальный интегратор;
- не редактор проектной документации.

Его роль:
- дешёвый генератор локального patch-кандидата.

Финальная ответственность:
- у пользователя;
- и у Codex как технического фильтра, интегратора и верификатора.
