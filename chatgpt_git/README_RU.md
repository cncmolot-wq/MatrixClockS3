# ChatGPT + GitHub Package

Этот каталог — отдельный комплект для внешнего обычного ChatGPT.

Его задача:
- дать внешний набор правил для ChatGPT
- вынести их в GitHub
- сделать ссылки стабильными
- не смешивать этот внешний пакет с живой внутренней документацией проекта

## Что лежит в папке

- `mc_core.v4.ini`
  внешний машинно-ориентированный профиль
- `CHATGPT_PROJECT_SETTINGS_RU.md`
  готовый текст для вставки в `Project Instructions`
- `CHATGPT_TASK_CAPSULE_RU.md`
  шаблон узкой задачи для внешнего ChatGPT
- `GITHUB_PUBLISH_STEPS_RU.md`
  пошаговая инструкция публикации

## Важно

Этот пакет не заменяет:
- `docs/project/README_RU.md`
- `docs/project/REFACTOR_PLAN_RU.md`
- `docs/project/PROJECT_EDIT_RULES.md`

Они остаются живым внутренним источником истины.

Этот пакет нужен для внешнего ChatGPT, который:
- не видит локальные файлы
- может читать только вставленный текст
- и иногда умеет ходить по GitHub-ссылкам

## Рабочая схема

1. Ты публикуешь содержимое этой папки в GitHub.
2. Подставляешь реальные ссылки в файлы пакета.
3. Копируешь текст из `CHATGPT_PROJECT_SETTINGS_RU.md` в настройки проекта обычного ChatGPT.
4. Для каждой узкой задачи используешь `CHATGPT_TASK_CAPSULE_RU.md`.
5. Результат приносишь обратно в проект.
6. Финальную интеграцию, проверку, сборку и синхронизацию выполняет Codex.

## Принцип

Внешний ChatGPT:
- не архитектор
- не финальный интегратор
- не редактор проектной документации

Его роль:
- дешёвый генератор локального patch-кандидата

Финальная ответственность:
- у пользователя
- и у Codex как технического фильтра и интегратора
